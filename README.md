# Логическая модель системы учета отходов

## Сущности:

1. **Район** (district)
   - `id` (PK) - идентификатор района
   - `name` - название района
   - `code` - код района

2. **Тип отходов** (waste_type)
   - `id` (PK) - идентификатор типа
   - `name` - название типа (стекло, пластик, бумага)
   - `code` - код типа

3. **Оператор вывоза** (waste_operator)
   - `id` (PK) - идентификатор оператора
   - `company_name` - название компании
   - `phone` - телефон
   - `email` - электронная почта
   - `license_number` - номер лицензии

4. **Контейнерная площадка** (collection_point)
   - `id` (PK) - идентификатор площадки
   - `address` - адрес
   - `coordinates` - координаты
   - `district_id` (FK) - ссылка на район
   - `is_active` - активна (да/нет)

5. **Акт вывоза** (waste_removal_act)
   - `id` (PK) - идентификатор акта
   - `point_id` (FK) - ссылка на площадку
   - `operator_id` (FK) - ссылка на оператора
   - `type_id` (FK) - ссылка на тип отходов
   - `removal_date` - дата вывоза
   - `weight_kg` - вес в кг
   - `vehicle_number` - номер транспорта
   - `driver_name` - водитель
   - `act_number` - номер акта

6. **Расписание вывоза** (removal_schedule)
   - `id` (PK) - идентификатор расписания
   - `point_id` (FK) - ссылка на площадку
   - `type_id` (FK) - ссылка на тип отходов
   - `schedule_type` - тип расписания (регулярное/по требованию)
   - `frequency_days` - частота в днях
   - `week_days` - дни недели
   - `preferred_time` - предпочтительное время

## Процессы:

- Операторы фиксируют акты вывоза отходов определенного типа с контейнерных площадок согласно установленному расписанию. Каждая контейнерная площадка привязана к району города и может иметь контейнеры для разных типов отходов.

## Выходные документы:

- Выдать отчет по общему весу вывезенных отходов каждого типа за месяц по районам города, отсортированный по убыванию веса

- Для заданной контейнерной площадки выдать график и частоту вывоза каждого типа отходов, отсортированный по дате
