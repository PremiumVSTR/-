# Логическая модель по диаграмме
## Сущности:
1. Район (id_района PK, название_района, код_района)
1. Контейнерная площадка (id_площадки PK, адрес, координаты, id_района FK, активна)
1. Тип отходов (id_типа PK, название_типа, код_типа)
1. Оператор (id_оператора PK, название_компании, телефон, email, номер_лицензии)
1. Акт вывоза (id_акта PK, id_площадки FK, id_оператора FK, id_типа FK, дата_вывоза, вес_кг, номер_транспорта, водитель, номер_акта)
1. Расписание вывоза (id_расписания PK, id_площадки FK, id_типа FK, тип_расписания, частота_дни, дни_недели, предпочтительное_время)

## Процессы:

Операторы фиксируют акты вывоза отходов определенного типа с контейнерных площадок согласно установленному расписанию. Каждая контейнерная площадка привязана к району города и может иметь контейнеры для разных типов отходов.

## Выходные документы:

Выдать отчет по общему весу вывезенных отходов каждого типа за месяц по районам города, отсортированный по убыванию веса

Для заданной контейнерной площадки выдать график и частоту вывоза каждого типа отходов, отсортированный по дате
